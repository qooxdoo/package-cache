## Project Purpose

This repository is the **qooxdoo Package Cache** — a data repository that maintains a JSON metadata cache of all available qooxdoo packages (libraries/plugins). It has no source code to compile and no tests to run. The cache (`cache.json`) is auto-generated by CI and should never be manually edited.

The cache is consumed by the qooxdoo Package Browser (https://qooxdoo.org/qxl.packagebrowser/) and by the `qx package` CLI commands.

## Triggering a Cache Update

The cache is updated automatically every 6 hours via GitHub Actions. To trigger a manual update:

```bash
# Via the trigger script (requires a GitHub token in GITHUB_TOKEN env var)
./trigger-update

# Or via GitHub CLI
gh workflow run update-cache.yml
```

The update workflow runs:
```bash
npx qx package update --verbose --search --all-versions --file cache.json
```

**Behavior after update:**
- New repositories detected → creates a PR for review before merging
- Only existing packages updated → auto-commits directly to master
- No changes → skips commit

## Architecture

### Key Files

| File | Purpose |
|------|---------|
| `cache.json` | Auto-generated data file — **do not edit manually** |
| `.github/workflows/update-cache.yml` | CI pipeline controlling how/when the cache updates |
| `trigger-update` | Bash script for manually triggering a cache rebuild via repository dispatch |
| `docs/index.md` | GitHub Pages frontend (JavaScript) for the Package Browser |

### cache.json Structure

```json
{
  "version": "2.1.0",
  "num_libraries": 583,
  "repos": {
    "list": ["owner/repo", ...],
    "data": {
      "owner/repo": {
        "description": "...",
        "releases": {
          "list": ["v1.0.0", ...],
          "data": {
            "v1.0.0": {
              "manifests": [...],
              "published_at": "ISO-8601"
            }
          }
        }
      }
    }
  }
}
```

### CI/CD Workflow Triggers

- Scheduled every 6 hours
- Push to master (unless only `docs/` changed)
- `workflow_dispatch` (manual)
- Repository dispatch event: `rebuild-cache`
